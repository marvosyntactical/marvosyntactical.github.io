Uncertainty Quantification using Bayesian Neural Networks¶

BNN Slides from last time
Our goal: Understand how to express different sources of uncertainty using BNNs, for the task of regression.

Theory - Aleatoric & Epistemic Uncertainty¶

(Hüllermeier & Wagemann (2020))

Uncertainty quantified by some model fit to data can come from two sources:

  1. ALEATORIC Spread/stochasticity/noise present in the data. Even with a perfect model, this uncertainty persists; it is IRREDUCIBLE.
  2. EPISTEMIC Uncertainty about which model is appropriate for the data. With more data, this type of uncertainty is REDUCIBLE.

Useful Mnemonics:

  • Aleatoric (Latin Alea: Dice): uncertainty due to data generated by dice-throw-like processes.
  • Epistemic (Greek episteme: knowledge): uncertainty due to a lack of knowledge.

How to distinguish these two?¶

  • There is no universal mathematical definition of these terms;
  • It depends on the class of models.
  • The presence of aleatoric uncertainty does NOT depend on the type of model, of course,
  • But whether and how we can distinguish it from epistemic uncertainty does.

In Neural Networks¶

  • Deterministic NNs (Maximum Likelihood Estimators) cannot quantify epistemic uncertainty.
  • Bayesian NNs can quantify the epistemic uncertainty through the part of the output variance caused by the weight distribution.
NNs applied to Regression¶

In the case of regression, we typically use our neural network with weights $\theta$ to parameterize a Gaussian Likelihood over some space: $$p_\theta(y|x) = \mathcal{N}(\mu,\sigma^2)$$ That is, we construct the architecture to learn the two parameters of a Gaussian distribution.

$Math$¶

When we make predictions $\hat{Y}\sim p_\theta(y|x) = \mathcal{N}(\mu,\sigma^2)$,
by the Law of total Variance,

$$\underset{\text{predictive variance}}{\mathbb{V}\text{ar}[\hat{Y}]} = \underset{\text{aleatoric uncertainty}}{\mathbb{E}_{p_\theta}[\sigma^2]} + \underset{\text{epistemic uncertainty}}{\mathbb{V}\text{ar}_{p_\theta}[\mu]}$$
Code¶

Sampling the BNN's weights $N$ times for one batch $x$ of $B$ datapoints, and output dimensionality $D$:

mu, sigma = bnn(x) # each of shape [N, B, D]
predictions = Normal(mu, sigma).sample() # [N, B, D] 
aleatoric = sigma.mean(dim=0) # [B, D]
epistemic = mu.var(dim=0) # [B, D]
predictive_variance = aleatoric + epistemic

We thus estimate the uncertainty of the prediction on each datapoint.

2.2 Architecture¶

We use a simple feedforward architecture with a Tanh() nonlinearity in all experiments.

It is however dependent on the kind of likelihood we chose:

  • Homoskedastic: Let the network only learn an x-dependent mean, and train a non-x-dependent standard deviation outside of the PyTorch network.
  • Heteroskedastic: Let the network learn both x-dependent mean and standard deviation.
In [10]:
# ========================= NETWORK DEFINITION =========================

class FF(nn.Module):
    def __init__(self, dim=50, heteroskedastic=False, min_scale=1e-4):
        """
        >>> net = FF()  # corresponds to:
        >>> net = nn.Sequential(nn.Linear(1, 50), nn.Tanh(), nn.Linear(50, 1))
        """
        super().__init__()
        
        self.fc1 = nn.Linear(1, dim)
        self.tanh = nn.Tanh()

        self.loc_head = nn.Linear(dim, 1)
        
        self.heteroskedastic = heteroskedastic
        if self.heteroskedastic:
            self.scale_head = nn.Linear(dim, 1)
            # NOTE: used to constrain variance to be > 0 and possibly smaller than some value
            self.scale_nonlinearity = lambda s: s.clamp(min=1e-6, max=max_likelihood_scale)
            self.min_scale = min_scale
In [11]:
    def forward(self, x):

        x = self.fc1(x)
        x = self.tanh(x)
            
        mean = self.loc_head(x)
        
        if not self.heteroskedastic:
            return mean
        else:
            scale = self.scale_head(x)
            scale = self.scale_nonlinearity(scale)
            # tyxe.likelihoods.HeteroskedasticGaussian expects network output to be 
            # a concatenation of mean (no constraint) and scale (> 0) along last (hidden) dimension
            mean_scale = torch.cat([mean, scale], dim=-1) # N x B x 2D
            return mean_scale

FF.forward = forward

# ============================= VARIATIONAL TRAINING SETUP =====================================

if inference in variational_infs:
    
    optim =  pyro.optim.ClippedAdam({"lr": lr, "lrd": .995})
    
    loss_kwargs = {} # kwargs for elbo
    total_steps = epochs * n_mini_batches
    kl_factors = []
    
    def kl_schedule(step_number: int):
        # is called every step (batch) to scale the KL term of the ELBO

        assert step_number > 0
        eps = 1e-6
        
        # clamped linear schedule (initially, keep elbo term very low (~= MLE), at the end, do full SVI)
        linear_schedule = lambda step: max(eps, step/total_steps - warmup_fraction)
        factor =  min(1.0, linear_schedule(step_number) / (1-warmup_fraction-full_fraction))

        # - The following schedule is from
        # - weight uncertainties in neural networks, blundell et al, Sec. 3.4 eq. (9):
        # n_steps = epochs * n_mini_batches
        # normalization = 1/(n_steps-1)
        # factor = max(1e-3, (n_steps - step_number)*normalization)

        kl_factors.append(factor)
        if monitor:
            n_run["train/kl_factors"].log(factor, step=step_number)
        return factor

    # logging
    progress_bar = tqdm(desc=inference, total=epochs)
    fig, axes = plt.subplots(ncols=2, nrows=4, figsize=big_figsize)
    plt.close(fig) # prevent separate figure from being shown
    hfig = display(fig, display_id=True)

    # train metric
    elbos = []
    # validation metrics
    nlls, rmses = [], []
    pred_vars, alea_uncs, epis_uncs = [], [], []
    scalar_metrics = { "nll": nlls, "rmse": rmses, "elbo": elbos }
    tensor_metrics = { "predictive": pred_vars, "aleatoric": alea_uncs, "epistemic": epis_uncs }

    def svi_callback(bnn, epoch, elbo):
        # every epoch:
        if monitor:
            n_run["train/elbo"].log(elbo, step=epoch)
        elbos.append(elbo)
        progress_bar.update()
        
        # validation:
        do_validation = epoch % valid_freq == 0
        if do_validation:               
            with valid_ctxt():
                rmse, nll, predictive_variance, aleatoric_uncertainty, epistemic_uncertainty, sampled_predictions, raw_predictions = tuple(map(
                    lambda _tnsr: _tnsr.cpu().detach(),
                    bnn.evaluate(
                        x_test.to(device=device), y_test.to(device=device), num_predictions=n_test_samples, aggregate=False
                    )
                ))
            
            # update validation metrics
            nlls.append(nll)
            rmses.append(rmse)
            pred_vars.append(predictive_variance)
            alea_uncs.append(aleatoric_uncertainty)
            epis_uncs.append(epistemic_uncertainty)

            plot(
                bnn,
                n_test_samples,
                aggregate=False,
                axes=axes,
                fig=fig,
                hfig=hfig,
                # alpha=epoch/epochs+1e-2,
                data_tuple=data_tuple,
                scalar_metrics=scalar_metrics,
                tensor_metrics=tensor_metrics,
                valid_ctxt=valid_ctxt,
                device=device
            )
            
            if monitor:
                n_run["valid/nll"].log(nll, step=epoch)
                n_run["valid/rmse"].log(rmse, step=epoch)
                n_run["valid/pred_var"].log(predictive_variance.mean(), step=epoch)
                n_run["valid/aleatoric"].log(aleatoric_uncertainty.mean(), step=epoch)
                n_run["valid/epistemic"].log(epistemic_uncertainty.mean(), step=epoch)
                n_run[f"img/valid_{epoch}"].upload(neptune.types.File.as_image(fig))
    
    fit_args = [loader, optim, epochs, svi_callback]
    fit_kwargs = {
        "closed_form_kl": mean_field,
        "num_particles": elbo_samples,
        "device": device,
        "jit": jit,
        "kl_schedule": kl_schedule if do_kl_annealing else lambda n: 1.0,
    }
    fit_kwargs.update(loss_kwargs)

    train_duration = epochs
    duration_unit = "epochs"
